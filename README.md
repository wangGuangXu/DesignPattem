## 如果编写程序的时候考虑的都是如何针对抽象编程而不是针对细节编程,即程序中所有的依赖关系都是终止于抽象类或接口,那就是面向对象的设计,反正就是过程化的设计.
## 面向对象原则：
### 单一职责原则：就一个类而言，应该仅有一个引起它变化的原因。
### 开放-封闭原则：对于扩展是开放的，对于更改是封闭的。
### 依赖倒置原则：抽象不应该依赖于细节,细节应该依赖于抽象。
### 里氏替换原则：子类必须能够替换掉它们的父类型
